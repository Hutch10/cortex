# Stage 1 Observer Field Guide

## Primary Question
Can ordinary users form a sufficiently accurate mental model of Vitalicast's preservation behavior to trust it with long-lived personal records?

## Observer Rule
Record the following, keeping them strictly separated:
1. Observed Behavior
2. Direct User Statement
3. Observer Interpretation
4. Open Question

## Explicit Prohibitions
Observers MUST NOT:
* Teach the user that records are immutable before testing their mental model.
* Explain addenda before observing the user's expectation.
* Tell the user exports prove ownership before asking what they think an export is.
* Praise a "correct" answer during the evaluation.
* Correct a misconception before the misconception has been recorded.
* Convert paraphrase into a direct quote.
* Treat silence as understanding.
* Treat task success as proof of correct mental model.

## Assistance
The observer may assist when necessary. When assistance occurs, record:
* Assistance given
* Point in session
* Pre-assistance misunderstanding
* Immediate recovery
* Whether understanding persists later
Do not hide intervention.

## Stage 1 Task Sequence (Neutral)
TASK 1 — CAPTURE
Ask the tester to create a record using the normal current Beta 2 experience.
After completion, use a neutral prompt:
"Tell me what you think Vitalicast recorded."
(Do not explain the intended answer first).

TASK 2 — PRESERVATION EXPECTATION
After the relevant current Beta 2 state is visible or experienced, ask:
"Tell me what you think happens to this record from here."
Where appropriate later ask:
"What would you expect to happen if you wanted to change or add something later?"
(Do not use terms like immutable, canonical, preservation grade, or append-only unless the tester introduces them).

TASK 3 — LATER CONTEXT / ADDENDUM
Use the current Beta 2 addendum behavior. Before teaching the interaction, establish the tester's expectation where practical.
After use ask:
"Tell me what you think happened to the original record."

TASK 4 — SEARCH / RETRIEVAL
Ask:
"Show me how you would find this again."
After retrieval ask:
"What do you think Vitalicast is doing when it shows you this result?"
(Do not ask leading questions like "Do you understand that search is deterministic?").

TASK 5 — EXPORT / OWNERSHIP
Use only the current Beta 2 export behavior actually available.
Ask:
"What do you think this export contains?"
and:
"What do you think you could do with it if you stopped using Vitalicast?"
(Do not claim offline or independent usefulness beyond current Beta 2 evidence).

TASK 6 — TRUST CALIBRATION
Ask neutral questions equivalent to:
"What do you believe Vitalicast can tell you about a record like this?"
"What do you think Vitalicast cannot know from this record alone?"
Record any belief that Vitalicast proves truth, provides medical validation, knows causality, guarantees completeness, understands all meaning, or cannot lose data as potential trust-calibration evidence. Do not argue with the tester during initial elicitation.
